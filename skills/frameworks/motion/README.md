# Motion skills

Animation and motion guidance for React applications using Motion and modern UI transition patterns.

Motion skills should keep animation meaningful, maintainable, performant, and easy to reason about.

## Philosophy

- Prefer meaningful motion over animation spectacle
- Prefer subtle transitions over distracting movement
- Keep animation ownership explicit
- Prefer composable animation primitives
- Avoid animation-driven architecture
- Optimize for responsiveness and perceived smoothness
- Respect accessibility and reduced-motion preferences

## Current Stack

Current preferred animation stack:

- Motion
- React
- Tailwind CSS

These are preferences, not hard requirements.

- **[motion-discipline](./motion-discipline/SKILL.md)** — Motion guidance focused on maintainable animation patterns, transitions, layout animation, gesture handling, and accessible motion systems.
