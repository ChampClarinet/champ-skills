# Next.js skills

Next.js App Router, server/client boundaries, rendering, routing, and maintainable full-stack React application guidance.

Next.js skills should keep rendering behavior explicit, data flow predictable, and client/server responsibilities easy to reason about.

## Philosophy

- Prefer Server Components by default
- Keep client boundaries intentional
- Prefer explicit data ownership
- Avoid unnecessary client-side state and effects
- Prefer framework-native patterns over custom orchestration
- Keep routing and rendering behavior easy to trace
- Optimize for maintainability and predictable execution flow

## Current Stack

Current preferred stack commonly includes:

- Next.js App Router
- TypeScript
- Tailwind CSS
- shadcn/ui
- Radix UI

These are preferences, not hard requirements.
