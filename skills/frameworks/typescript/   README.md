# TypeScript skills

TypeScript architecture, type design, and maintainable type-system guidance.

TypeScript skills should keep types explicit, understandable, maintainable, and easy to evolve.

## Philosophy

- TypeScript-first
- Prefer explicitness over type wizardry
- Prefer maintainable inference
- Prefer readable types
- Keep type ownership predictable
- Avoid unnecessary complexity in the type system
- Use types to improve correctness and maintainability, not to impress

## Skills

- **[type-discipline](./type-discipline/SKILL.md)** — TypeScript guidance focused on maintainable typing, explicit contracts, safe inference, modular type organization, and long-term readability.
