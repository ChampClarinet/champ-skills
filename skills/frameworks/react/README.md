# React skills

React-specific component structure, TypeScript usage, state ownership, hooks discipline, and UI implementation guidance.

React skills should keep components explicit, maintainable, modular, and easy to reason about.

## Philosophy

- Use TypeScript-first React
- Prefer predictable component boundaries
- Keep state ownership clear
- Prefer readable JSX over clever abstraction
- Use local state before global state when possible
- Keep effects intentional
- Keep styling consistent with the project stack
- Avoid premature abstraction and unnecessary memoization

## Current stack preference

- TypeScript
- Tailwind CSS
- shadcn/ui
- Radix UI when needed

These preferences reflect the current project stack. Follow project conventions if they differ.
